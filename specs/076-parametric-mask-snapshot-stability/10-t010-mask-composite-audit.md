# 10 - T-010 Mask/Composite Audit

## Task

T-010 Body vs overlay mask-path audit.

## Summary

The renderer currently applies masking through two different constructions:

1. Base body path uses element mask definition generated by `buildElementMaskDef` and then applied to silhouette/body path.
2. Overlay texture/gradient/material paths use additional per-layer masks generated from `resolveClipMaskBody` and separate mask tags.

This split can create divergence under heavy stacking and clip inheritance edge cases.

## Mismatch Inventory

### M-001 Dual masking strategy between body and overlays

Body branch and overlay branch do not fully share identical mask-source selection logic. This increases chance of branch divergence where overlays remain visible while body is attenuated.

### M-002 Overlay mask tag does not explicitly set maskUnits

Layer-level masks set maskContentUnits but do not set maskUnits explicitly. In transformed or complex contexts this can increase coordinate interpretation inconsistency risk.

### M-003 Clip target defaults can self-target first

Editor default clip target selection uses selected element name first. Renderer later guards explicit self-target by falling back, but the input state pattern can still produce ambiguous user intent and unstable inheritance behavior.

### M-004 Inheritance target resolution is string-name dependent

Clip target uses registry lookup by element name and previous element name. Missing or stale naming sequences fall back to local UV input body, which can vary from expected target semantics in stacked setups.

## Intended Fix Direction for T-011

1. Align body and overlay effective mask semantics at compositing boundary.
2. Make layer mask definitions explicit and consistent for units configuration.
3. Harden clip target validation to avoid ambiguous self/inherit edge cases.
4. Preserve existing procedural path behavior for valid states.
